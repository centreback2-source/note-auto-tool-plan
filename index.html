note記事作成・投稿支援アプリ
仕様解説

本アプリは、note向けの記事作成、Gemini APIによる文章支援、アイキャッチ生成、投稿前チェック、note投稿画面への入力補助を、ユーザーのローカル環境中心で安全に提供する制作支援ツールです。

対象仕様: 要件定義・詳細仕様書 v2.0 MVP: Windows版 方式: ローカルWebアプリ型 AI: Gemini API / ユーザー自身のキー 投稿: 公開直前停止

1. まず押さえる全体像

このアプリは「記事を自動で公開するツール」ではありません。記事作成から投稿画面への入力までを支援し、最後の公開判断はユーザー確認に戻す設計です。

作る

テンプレート、本文インポート、Gemini APIを使って、note記事の下書き・見出し・タイトル・タグ・有料記事構成を作成します。

整える

アイキャッチ生成、無料部分と有料部分の分割、価格・返金設定、販売導線チェック、読みやすさ確認を行います。

投稿を補助する

ユーザーPC上のブラウザを操作してnote投稿画面へ入力します。標準では公開ボタンの手前で必ず停止します。

✓
設計の中心方針

記事本文、noteログイン状態、Gemini APIキーをできる限りユーザーのPC内に置き、クラウド側に秘密情報を集めない構成にします。

項目方針理由
初期提供形態Windows向けローカルWebアプリ型デスクトップアプリnote投稿支援、APIキー管理、記事保存をローカルで安全に扱いやすい。
Gemini APIユーザー自身のAPIキーを利用運営者側のAPI費用・レート制限・キー漏洩リスクを抑える。
note投稿ローカルPlaywrightで投稿画面へ入力公式公開APIに依存せず、ユーザーのブラウザ操作を補助する。
公開操作標準では自動で押さない誤投稿、価格設定ミス、有料ラインミスを防ぐ。
モバイル対応まずは記事作成・コピー補助中心iOS/AndroidではPCと同じブラウザ自動操作が難しいため。

2. システム構成

exe化したPythonアプリがユーザーPC上でローカルサーバーを起動し、ユーザーは普段のブラウザから画面を操作します。外部に出る通信は主にGemini APIとnoteサイトです。

ローカルサーバー

exe起動時に 127.0.0.1 のランダムポートで起動します。外部ネットワークから接続できる 0.0.0.0 では起動しません。

ブラウザUI

見た目と操作感はWebアプリに近くします。将来のPWA・モバイル展開に備え、UIと処理を分離します。

投稿支援

note操作はユーザーPC上のPlaywrightブラウザで実行します。noteのログイン状態をクラウドに預けない方針です。

3. 利用の流れ

画面はウィザード形式を基本とし、1画面1目的で進めます。投稿前には必ず確認画面を挟みます。

1. 記事タイプ無料記事 / 有料記事を選択。目的と読者も入力。
2. 作成モードA: テンプレート、B: インポート、C: Geminiを選択。
3. 本文作成タイトル、見出し、本文、タグ案を作成・編集。
4. 有料設計有料ライン、価格、返金設定、購入価値を確認。
5. 画像生成Pillowでテンプレート型アイキャッチを作成。
6. 投稿前チェック不足項目、誤設定、公開リスクを確認。
7. note入力Playwrightで投稿画面へ入力し、公開直前で停止。
!
標準では公開しない

アプリは投稿画面への入力を支援しますが、MVPでは公開ボタンを自動で押しません。公開判断はユーザー確認に戻します。

4. 記事作成モード

APIなしでも使える基本機能を持ち、必要なユーザーだけGemini APIを使える構成にします。

A

テンプレート生成

入力項目をもとに、定型テンプレートで記事の骨子と本文を生成します。API不要で無料利用できます。

API不要MVP必須安定
B

本文インポート

既存の文章、メモ、Markdownなどを取り込み、note向けに整えます。既に原稿があるユーザー向けです。

API不要編集中心既存原稿
C

Gemini API連携

ユーザー自身のGemini APIキーを使い、文章生成、リライト、タイトル案、有料ライン提案などを行います。

任意ユーザーAPIキーモデル自動切替
モード主な用途API費用位置づけ
Aモード短時間で記事のたたき台を作る不要基本機能
Bモード既存原稿をnote向けに整える不要基本機能
CモードAIで生成・改善・販売設計を強化するユーザー負担任意オプション

5. Gemini API設計

CモードではGemini APIを前提にします。アプリ側のModel Routerが、処理内容・文字量・有料記事かどうか・ユーザー設定を見てモデルを選びます。

ユーザー向け表示

画面上では「速度優先」「品質優先」「無料枠優先」「自動」「手動固定」のような分かりやすい選択肢を出します。生のモデル名は詳細設定に置きます。

エラー時の扱い

レート制限や利用上限に当たった場合は、軽量モデルへの再試行、処理分割、テンプレート生成への戻しを行います。

処理難易度初期モデル候補補足
タイトル案・タグ案低Flash-Lite系高速・低コストで十分な処理。
要約・軽いリライト低〜中Flash-Lite系 / Flash系文字量で切り替える。
本文生成・構成案中Flash系Cモードの標準候補。
有料記事の販売設計高Flash系 / Pro系価格、無料部分、有料部分の納得感を見る。
長文整合性チェック高Pro系長文・複雑な確認向け。

i
モバイル版の位置づけ

iOS/Androidでは、まず「記事作成・投稿用コピー補助・画像保存・note起動」を価値にします。PC版と同じ自動投稿体験を初期目標にしません。

7. note投稿支援

投稿支援は、ユーザーPC上で起動したブラウザをPlaywrightで操作する方式です。noteの非公式APIに直接リクエストする実装は行いません。

アプリが行うこと

投稿画面を開き、タイトル、本文、アイキャッチ、タグ、無料/有料記事設定、価格、返金設定、有料ラインの入力を補助します。

標準では行わないこと

公開ボタンの自動クリック、note非公式APIの直接利用、クラウド上でのnoteセッション保管は行いません。

8. 有料記事の安全設計

有料記事では、無料部分・有料部分・有料ライン・価格・返金設定を明確に扱います。誤投稿防止のため、有料記事では確認を強めます。

無料部分

読者が購入前に読める範囲。問題提起、得られる価値、信頼材料、本文の一部を配置します。

有料ライン

有料部分

購入後に読める範囲。具体的な手順、テンプレート、事例、深い解説などを配置します。

境界チェック

有料ラインは段落間にのみ配置します。無料部分と有料部分の境界が曖昧な状態では投稿に進めません。

価格チェック

価格が未設定、異常値、意図と違う値の場合は警告します。投稿前画面で再確認します。

返金設定チェック

返金可否の設定を明示し、確認漏れがないようにします。設定内容は投稿前に表示します。

!
有料本文の扱い

有料部分の全文はログに出しません。エラー発生時も、必要最小限の情報だけを記録します。

9. セキュリティ境界

このアプリでは、秘密情報をできる限りローカルに閉じ、ログに出さず、クラウドに集めないことを重視します。

守るもの

Gemini APIキー、noteログイン状態、Cookie、記事本文、有料本文、アイキャッチ画像、投稿履歴、エラースクリーンショット。

やってはいけないこと

APIキーの平文保存、ログ出力、クラウドへのnoteセッション保存、外部から見えるローカルサーバー起動、公開ボタンの無確認クリック。

対象保存先・扱い注意点
Gemini APIキーOS資格情報ストアDB、ログ、スクリーンショットに出さない。
noteログイン状態ローカルPlaywrightプロファイルクラウド保存しない。削除機能を用意する。
記事本文SQLite / ローカルファイル保存先を明示。バックアップ・削除を用意する。
有料本文ローカル保存ログ出力・エラー送信・全文表示の扱いを制限する。
ローカルサーバー127.0.0.1 のみ0.0.0.0 で起動しない。起動ごとのトークンを使う。
ローカルサーバーは127.0.0.1のみ同じLAN上の他端末から触れないようにする。
起動ごとの認証トークンURLにランダムトークンを付け、不要なアクセスを防ぐ。
APIキーをログに出さない例外ログ、通信ログ、画面キャプチャにも含めない。
有料本文全文をログに出さないエラー調査は記事ID、ステップ、短い要約程度に留める。
公開操作はユーザー確認公開直前停止を標準仕様にする。
削除・リセット導線認証状態、スクリーンショット、ログ、記事データの削除機能を用意する。

10. データ保存

アプリの基本データはローカルSQLiteとローカルファイルに保存します。将来クラウド同期を追加する場合も、noteセッションや秘密情報は同期対象にしません。

i
将来のクラウド同期

同期対象にするのは、テンプレート、下書き、設定の一部、ライセンス情報などに限定します。Gemini APIキーやnoteセッションは同期対象外にします。

12. レビュー観点

設計レビューでは、以下の判断が合意できているかを確認します。ここが固まると、実装のブレが小さくなります。

MVPはWindows版で開始するmacOS/Linux/Web/モバイルは段階展開にする。
ローカルWebアプリ型を採用するexeがローカルサーバーを起動し、ブラウザUIで操作する。
Gemini APIはユーザー自身のキーを使う開発者のAPIキーを同梱しない。
Geminiモデルは自動切替にする処理難易度とユーザー設定に応じてModel Routerが選ぶ。
note投稿は公開直前で停止する標準では公開ボタンを自動クリックしない。
note非公式APIを直接使わないユーザーPC上のブラウザ操作補助として実装する。
有料記事は確認を強める有料ライン、価格、返金設定を投稿前に必ず確認する。
モバイル版は作成・補助中心にするiOS/Androidでの完全自動投稿を初期必須要件にしない。

補足: 用語ミニ解説

ローカルWebアプリ型

アプリ自体はPCで動きますが、画面はブラウザで表示する方式です。Webアプリのように操作でき、データはPC内に置きやすい構成です。

Playwright

ブラウザ操作を自動化する仕組みです。このアプリではnote投稿画面への入力補助に使います。

OS資格情報ストア

Windows Credential Manager、macOS Keychainなど、OSが用意する秘密情報保存の仕組みです。

Model Router

AIに依頼する内容の難しさに応じて、軽量モデル・標準モデル・高品質モデルを自動で選ぶアプリ内の部品です。
