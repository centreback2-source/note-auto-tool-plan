note 自動投稿ツール — 全体設計図 Web版

Architecture Overview / 開発者・編集チーム向けレビュー資料

Webアプリ（URLアクセスのみ） Next.js + TypeScript Claude API (Anthropic) DALL-E 3 (OpenAI) Playwright（投稿サーバー） Windows / Mac / iOS / Android 全対応

このツールでできること

🌐

インストール不要

URLにアクセスするだけで利用開始。PC（Windows/Mac）・スマホ（iOS/Android）すべてのブラウザで動く。

✍️

記事を全自動執筆

ヒアリングからジャンル選定・タイトル採点・本文（10,000字以上）・アイキャッチ画像まで一気通貫で生成する。

📤

note へ自動投稿

サーバー上でブラウザを自動操作してタイトル・本文・アイキャッチ・価格を設定。デフォルトは下書き保存で安全。

APIキー取得フロー ─ ユーザーの初期セットアップ

方針: APIキーの取得（アカウント作成・クレカ登録）はユーザー自身が行う。アプリは「取得手順のガイド」と「入力欄」を提供する。ガイド画面にはスクリーンショットを同梱し、Anthropic / OpenAI の管理画面の操作場所を視覚的に示す。
🤖 Anthropic（Claude）10記事で約200〜350円
  1. console.anthropic.com にアクセス
  2. アカウント作成（クレジットカード登録）
  3. 「API Keys」→「Create Key」
  4. sk-ant-... で始まるキーをコピー
  5. アプリの入力欄に貼り付け
画面で操作箇所をスクショ付きでガイドする。
🎨 OpenAI（DALL-E 3）10記事で約100〜150円
  1. platform.openai.com にアクセス
  2. アカウント作成（クレジットカード登録）
  3. 「API Keys」→「Create new secret key」
  4. sk-... で始まるキーをコピー
  5. クレジット残高を最低 $5 チャージ（DALL-E 3 利用に必要）
  6. アプリの入力欄に貼り付け
売れる記事にはアイキャッチが必須のため、OpenAI キーも必須としている。
費用に関する明示: セットアップ画面の冒頭で「このツール自体は無料ですが、AI の利用料金は別途かかります」「10記事の生成でおよそ 300〜500円 が Anthropic と OpenAI に対して発生します」を必ず提示する。あとからの「課金されると思っていなかった」トラブルを防止する。

各フェーズの詳細

Phase A ─ ヒアリング対話　（Claude Haiku 4.5）
  • 起動セリフ固定（変更不可）。AIキャラクターは「横から共有する仲間」のスタンス
  • 1回1問ルール厳守。複数質問は禁止
  • 相手の言葉を「オウム返し＋意味の言語化」してから次へ進む
  • 直接聞かず体験を掘る迂回質問で6項目を引き出す
  • ①発信ジャンル ②ターゲット読者 ③深い悩み ④最悪の未来 ⑤解決策 ⑥最高の未来 が揃ったら自動で Phase B へ
  • 会話ログは localStorage に保存（途中再開可能）
Phase B ─ ジャンル・ペルソナ確定　（Claude Sonnet 4.6）
  • 会話ログから100ジャンルリストへ自動マッピング → 最適ジャンルを1つ自動確定
  • ペルソナ8項目（名前・年齢・職業・悩み・購入理由 等）を1回のAPIコールで出力
  • ジャンルを10区画に分割（カバレッジマップ生成）
  • 有料記事1本 + 無料記事9本の割り当てを自動決定
  • ユーザーの承認なしにそのまま Phase C へ移行（確認不要・自動進行）
  • 結果は画面に表示する
Phase C ─ 10記事の連続生成　（全自動）
  • 各記事について 7ステップを順番に実行（次のセクション参照）
  • 有料記事: 10,000字以上。無料記事: 3,000〜5,000字
  • 品質チェックで NG なら最大2回まで自動リトライ
  • 1本生成完了ごとに localStorage の coverage_map.json を更新
  • 10本完成で完了通知
  • 途中でブラウザを閉じても、次回起動時に続きから再開できる

Phase C ─ 1記事あたりの7ステップ（すべてブラウザ内で実行）

1

タイトル10案生成 + 採点 + 自動採用

30文字前後のタイトルを10案生成。ターゲット明確性・クリック強度・内容一致性など5軸で100点満点採点。最高得点を自動採用。

Claude Sonnet 4.6 ブラウザ内
2

リサーチ（web検索で素材収集）

web_search ツールを有効化し、現状・問題点・失敗事例・成功事例・海外研究など8項目の素材をURL付きで収集する。

Claude Sonnet 4.6 web_search 有効 ブラウザ内
3

結起承転 構成生成

有料記事: 無料ゾーン（結→起→承→転の渇望を作る）＋「ここから有料」マーカー＋有料ゾーン（具体手順）。無料記事: 結起承転で完結。

Claude Sonnet 4.6 ブラウザ内
4

本文生成

冒頭は読者の心の声を「」で3〜4つ。本文はです・ます調統一、同語尾3連続禁止、感情6:理解4の比率、情景描写を含める。AI語禁止。

Claude Sonnet 4.6 ブラウザ内
5

品質チェック

構成チェック・文体チェック・読者チェック・文字数チェックの4カテゴリ全項目を確認。NGがあれば最大2回まで自動リトライ。

Claude Haiku 4.5 ブラウザ内
6

アイキャッチ画像生成（必須）

8ステップ自動判断（タイトル構造分類→人物使用判断→ペルソナ性別年齢参照→色設計）でDALL-E 3用英語プロンプトを自動生成・画像生成。売れる記事には必須のため省略不可。

DALL-E 3 (OpenAI) ブラウザ内
7

ブラウザに保存

記事Markdown・画像Base64・カバレッジマップを localStorage に保存。次の記事へ進む。10本完了で完了通知を表示。

localStorage ブラウザ内
📤

note 投稿（ユーザーが「投稿」を押したとき）

ブラウザから記事 + 画像 + note認証情報を投稿サーバーへ送信。サーバー上でPlaywright が note.com を操作。タイトル/本文/タグ/価格/アイキャッチを入力。有料記事は境界線を設定。

Playwright (サーバー) サーバー側で実行

セキュリティ設計

🔑

APIキーはサーバーを経由しない

Anthropic / OpenAI のSDKをブラウザ内で直接利用し、APIキーは各プロバイダのサーバーへ直接送信される。我々のサーバーには一切届かない。

💾

note認証情報の永続化なし

投稿サーバーは note ID/PW をDB・ディスクに一切書き込まない。プロセスメモリでのみ扱い、投稿処理完了後に明示的に破棄する。

🫣

ログのマスキング

FastAPI のミドルウェアでリクエストボディ中のパスワードを自動マスク。アクセスログにメールアドレス・パスワードを残さない。

🔒

HTTPS 必須 / CORS 制限

HTTP 経由のアクセスは強制リダイレクト。投稿サーバーはフロントエンドのドメインからのみアクセス可能（CORS）。

⏱

レート制限

同一IPからの投稿リクエストは10件/時間程度に制限。悪用や負荷集中を防ぐ。

📂

オープンソース化で透明性確保

フロントエンド・投稿サーバーともにGitHub公開。「何がどこに送信されるか」をユーザーが自分のコードリーディングで検証できる。

実装着手前に決めたい事項

☁️

投稿サーバーのデプロイ先確定

Render 無料枠（コールドスタートあり）か、Fly.io（月$5・安定）か、自前 VPS か。初期は無料枠で開始し、利用者が増えたら有料へ移行する案も。

🔍

note 有料境界線 UI の事前検証

note の編集画面で「ここから有料」ボタンのPlaywright操作が安定するかを、実際のブラウザで手動確認が必要。note の UI 変更頻度が高いため要事前検証。

📄

ナレッジドキュメントの取り扱い

フロントエンドJSにバンドルすると GitHub 公開時に全文露出する。サーバー側で保護するか、そもそも公開を許容するか、設計判断が必要。

📱

スマホ対応のスコープ

投稿まで含めて完全対応するか、生成だけスマホ・投稿はPC推奨にするか。スマホの localStorage 容量制限（5MB前後）も考慮要素。

⚖️

利用規約・プライバシーポリシー

「note認証情報はメモリで処理し保存しない」「アクセスログにIP保持」などを明示。法務確認の要否を判断する。

💡

ヒアリングモデルの実機検証

Haiku 4.5 だけで対話品質（オウム返し・空虚な褒め禁止・寄り添いの距離感）が担保できるか、実機でのテストが必要。
